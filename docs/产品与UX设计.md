# 职能沟通翻译助手 - 产品界面规划与 UX 设计

## 一、Sequential 分析步骤

### Step 1：核心功能与用户需求

- **主要用户**：产品经理、开发工程师（需在需求讨论中互相理解对方表述）。
- **核心价值**：把「产品语言」翻译成「开发语言」，或把「技术方案」翻译成「业务语言」。
- **必须支持的界面功能**：
  1. **多轮对话**：连续多轮输入与翻译结果，形成会话历史。
  2. **身份/翻译方向切换**：当前以「产品→开发」或「开发→产品」进行翻译，可随时切换。
  3. **模型切换**：选择不同大模型（如 OpenAI、Claude、国内模型）。
  4. **模型参数设置**：Temperature、Max Tokens 等可调。
  5. **Agent 评测**：对翻译质量或 Agent 行为做评测（预设用例、运行、查看结果）。

### Step 2：信息架构（IA）

- **一级结构**：对话（首页）、设置、评测。
- **对话页**：承载多轮对话、身份切换、当前模型展示与快捷切换入口。
- **设置页**：模型列表、当前模型选择、参数表单（temperature、max_tokens 等）。
- **评测页**：评测用例列表、执行评测、结果展示（通过/失败、得分或评语）。

### Step 3：关键界面定义

| 界面       | 路径/文件    | 职责 |
|------------|--------------|------|
| 主入口     | index.html   | 品牌/简介 + 入口按钮进入「对话」或直接作为导航壳。 |
| 对话       | home.html    | 多轮对话区、身份切换、模型快捷切换、输入框、流式结果展示。 |
| 设置       | settings.html| 模型选择、模型参数（temperature、max_tokens 等）。 |
| Agent 评测 | evaluate.html| 用例列表、运行评测、结果列表（状态、耗时、简要结果）。 |

### Step 4：核心交互逻辑

- **对话页**：选择方向（产品→开发 / 开发→产品）→ 输入内容 → 发送 → 流式显示翻译 → 可继续追问或切换方向后再发。
- **设置页**：选择模型 → 修改参数 → 保存（写入本地或后端）；对话页使用的模型与参数与设置页一致（或设置页为全局默认）。
- **评测页**：选择用例（或全选）→ 执行评测 → 后端跑完返回结果 → 展示通过率、每条结果。

### Step 5：视觉与真实感

- 模拟桌面端网页：最大宽度约 1200–1400px，圆角卡片、柔和阴影。
- 可滚动区域隐藏滚动条（CSS 处理）。
- 使用真实图片（如 Unsplash 协作/办公场景）增强真实感。
- 统一使用 Tailwind CSS + FontAwesome，风格现代、简洁。

## 二、界面与路由对应

- `index.html`：主入口，导航到 home / settings / evaluate。
- `home.html`：对话（多轮、身份切换、模型切换入口）。
- `settings.html`：模型与参数。
- `evaluate.html`：Agent 评测。

所有页面顶部或侧边保持统一导航，确保可从任意页跳转至其他页。
